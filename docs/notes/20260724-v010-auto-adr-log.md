(PHPStan-grade checker; ROADMAP M1+M2, release gate M3): significant
design items encountered mid-run are commissioned to Fable subagents
as they arise; the resulting ADRs are committed immediately with
binding effect so implementation never blocks, and **every such ADR is
logged here** (number, one-line thrust, key decisions, open questions)
for the owner's batch review. Ratification states: PENDING → RATIFIED
/ AMENDED / VETOED (a veto reverts the ADR and its dependent slices).
One file for the whole run; newest entries appended at the bottom.

### ADR-0049 + ADR-0052 amendment batch — audit-hardened ladders — 2026-07-24

Thrust: fold the pre-implementation soundness audit
(`20260724-adr0049-0052-soundness-audit.md`, G1–G8/M1–M5) into
normative ladder legs — appended amendment sections A1–A11 on ADR-0049
and points 1–2 on ADR-0052 (§5/§11), plus in-place *amended* markers on
the three over-claiming sentences — so S2–S6/N4 inherit the legs from
the ADRs, not from an out-of-band note.

Key decisions: `$this` is membership, not exactness (`class_exact` bit
on the heap object; inexact receivers route to §8 or silence);
dam-freedom scoped to unconditional top-level homonym-free chains
(`conditional` flag re-dams; runtime-surface homonym ⇒ Unknown;
alias/decl collision ⇒ Ambiguous); enums Unknown for method-absence
and arity until fully lowered, with the reflect slice's gate carrying
the enum-lowering precondition; descendant closure over declarations
with class_alias/interface edges and an anonymous-class closure
obstacle; bare relative includes are dam sites (only
absolute/`__DIR__` literals provable; `[runtime] include-path` the
opt-in); SAPI-recorded existence posture, curated never-Absent set,
`[runtime] sapi`; `offset.missing` read-context whitelist
(isset-family / by-ref / unresolved-callee positions silent);
Relative ref kind required before S4; monkey-patch extensions
(uopz/runkit7/Componere) silence the whole family; read-side key
lookup shares the engine key canonicalization; catalog-backed is-a
demoted to Unknown for arm deletion under PHP-minor version skew.
ADR-0052: the derivation clause (a derived fact's stratum is the
minimum over its inputs, through folds, composition, heap writes, and
joins — emission-time premise checking alone is insufficient); N2
reordered from order-free to a hard prerequisite of S2–S6 (the
assert-provenance leakage is live in `apply_assert_to_var`).

Divergence from the audit, recorded in A2(ii): the audit's pre-reflect
approximation for the homonym leg ("FQN present in the builtin
hierarchy table ⇒ silent") is rejected as the normative form — the
leg's load-bearing direction is permissive, and absence from the
352-entry table proves nothing (the ADR's own catalog-is-never-an-
absence-oracle dictum). The leg instead consumes the S1 sidecar
existence surface (an existence question, answerable before the
reflect slice); table presence survives only as an extra silence.
Consequence accepted: sidecar-less runs keep `call.undefined-method`
Unknown-silent — the ADR-0004 sound subset now covers the flagship.

Open questions: whether any future completeness claim over the
runtime boot surface can partially reopen sidecar-less method-absence
(currently: no); M1 per-minor hierarchy table generation vs the v1
blanket demotion. Two of the original four were answered empirically
the same day (below).

Answered 2026-07-24 (verified `php -r` on PHP 8.5.8, orchestrator):

- **Compound-assignment read-half**: `$a[0]++`, `$a[0] .= "x"`, and
  `$a[0] += 5` on a missing key ALL emit `Warning: Undefined array
  key 0` (the read half fires; the write half then creates the key).
  `$a[0] ??= 5` is SILENT (isset semantics). Consequence for the §7/A7
  whitelist: compound assignments are read-context **eligible**;
  `??=` joins the isset-family silence leg.
- **Include precedence**: a bare relative literal (`include
  "target.php"`) resolves include_path → calling script's directory →
  CWD, in that order (all three positions verified by elimination).
  A `./`-prefixed literal (`include "./target.php"`) resolves against
  **CWD, not the script directory**. Consequence: the A5 wording
  ("only absolute and `__DIR__`-anchored literals are provable") is
  exactly right — `./` prefixing does NOT anchor to the including
  file, so `./`-literals stay Unproven dam sites; any future
  `[runtime] include-path` refinement must model the three-step order,
  and directory-relative belief (the pre-amendment `resolve_from`
  behavior) is wrong in both directions.

Status: RATIFIED (2026-07-24 batch; owner may re-review later).

### ADR-0053 The dump surface — 2026-07-24

Thrust: issue #17 (owner-requested, v0.1.0) — port the
`PHPStan\dumpType()` family and make `var_dump()` report inferred
facts by default; a dump is requested introspection, an ADR-0048
position query answered during the batch walk and rendered by the
honesty machinery.

Key decisions: a fourth registry layer `debug` (the ADR-0050 §1
by-ADR-only extension), three ids mapped onto the ADR-0052 carriers —
`debug.type` (best fact: value / exact-class / Member / contract arms
/ honest `unknown`), `debug.phpdoc-type` (contract-arm view),
`debug.var-dump` (one report per argument); exit split by authorship
of the question — the explicit pair is fail-level, fixed (a committed
`PHPStan\dumpType()` call is a runtime-fatal undefined function, so
the PHPStan remove-before-commit posture coincides with the zero-FP
identity), `debug.var-dump` is warn-level structurally exit-neutral
(measured ~63 live monorepo sites; quiet-default identity forces it,
and no promote-to-fail channel exists — fail-on-var_dump is lint,
refused per ADR-0017); explicit pair profile-inert, `debug.var-dump`
profile-disableable (named opt-out coexists with the owner's
default-ON), all three exempt from all three suppression channels and
the baseline; `PHPStan\` pair recognized unconditionally by resolved
FQN (reserved compat vocabulary, definition-insensitive), `var_dump`
trigger is resolution-sensitive under PHP's fallback rule (six
enumerated legs; namespaced-homonym or Unknown existence ⇒ silence);
recognized dump FQNs excluded from S4 `call.undefined-function`
(no double report); runtime semantics untouched (var_dump keeps its
effect coloring; dumpType stays a conservative unresolved call);
rendering through the N1 normalizer plus a plain-text arm spelling
moved into steins-contract shared with `annotate` (byte-equal parity
test; `(asserted)` stratum marker; `unknown` never guesses);
fp-gate excludes the debug layer from every counter with a
byte-identity assertion; assertType deferred to its own slice with
the name-keyed recognizer binding the consumption point open. Slices
D1 (lane groundwork, S1 pattern) → D2 (spelling extraction + annotate
rewire, byte-identical docblock output) → D3 (explicit pair + parity
+ fixture matrix) → D4 (var_dump + corpus judgment), D3/D4 sequenced
after the in-flight N2 (stratum input) and S2 (shared walk region).

Open questions: whether `annotate` should mark dump sites in the
margin (pure presentation, deferred); whether SARIF ever carries the
debug layer (default: omitted); the assertType harness's
spelling-compat contract (quarantined to its own slice by design).

Status: PENDING.
