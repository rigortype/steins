P1 stub-vs-runtime cross-check (PHP 8.5.8 cli, /opt/homebrew/bin/php)
php-src commit 6bc7c26cf67a9480b5ef9d6191aebe87fa931183

Method: stub declares DIRECT extends/implements; runtime class_implements()
returns the TRANSITIVE closure. Consistency holds iff closure(direct edges) ==
runtime set. Interface->interface edges (IteratorAggregate->Traversable,
Iterator->Traversable, SeekableIterator->Iterator, RecursiveIterator->Iterator)
are all present in hierarchy.toml, so the closure is reproducible.

                 Serializable,Countable

                 RecursiveIterator

                 Countable,JsonSerializable

VERDICT: no missing edges. Every runtime interface is derivable from the
recorded direct edges via transitive closure. The only "discrepancy" is by
design: stubs = direct edges, runtime = closure. Steins' oracle must compute
the closure (it already walks builtin_class_supers transitively), so the
direct-edge table is the correct feed. Nothing to patch.
